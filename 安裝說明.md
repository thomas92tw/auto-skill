# Auto-Skill 安裝說明

## 安裝日期
2026-02-10

## 安裝方式
使用 Git Submodule 方式安裝到 Antigravity_System

```bash
git submodule add https://github.com/thomas92tw/auto-skill.git skills/auto-skill
```

## 核心功能

### 1. 自我進化的知識積累系統
- **越用越強**：每次對話自動檢索過往經驗,調用最佳解法
- **跨技能經驗層**：記錄使用特定技能時的成功參數與避坑指南
- **主動式經驗捕獲**：任務完成時主動詢問是否記錄經驗
- **結構化知識存儲**：JSON 索引 + Markdown 內容,人類可讀、機器好懂

### 2. 運作邏輯 (5步循環)
1. **關鍵詞指紋**：從對話中提取核心關鍵詞
2. **話題切換偵測**：判斷是否開啟新話題
3. **經驗讀取**：檢查技能經驗庫中的踩坑紀錄
4. **知識庫檢索**：比對索引,載入最佳實踐
5. **主動記錄**：任務完成時執行經驗寫入

### 3. 檔案結構

```
auto-skill/
├── SKILL.md                    # 主要指令文件
├── knowledge-base/             # 通用知識庫
│   ├── _index.json            # 關鍵詞索引
│   ├── frontend-dev.md        # 前端開發
│   ├── backend-dev.md         # 後端開發
│   ├── writing.md             # 寫作技巧
│   ├── design-layout.md       # 設計排版
│   └── workflow.md            # 工作流程
└── experience/                 # 技能經驗庫
    ├── _index.json            # 技能索引
    └── skill-*.md             # 特定技能經驗
```

## 全局規則整合

已在 `~/.gemini/GEMINI.md` 加入任務啟動協議：

```markdown
## 任務啟動協議 (強制)

* 當開啟新任務或觸發任何技能時,必須先讀取並執行 auto-skill 技能的 SKILL.md。
```

## 使用方式

### 自動運作
- 不需要手動呼叫,auto-skill 會在背景自動運作
- 每次對話都會自動檢查知識庫與經驗庫
- 任務完成時會主動詢問是否記錄經驗

### 記錄經驗
當 AI 詢問「是否記錄經驗」時:
- 同意 → 經驗會被寫入對應的知識庫或經驗庫
- 拒絕 → 本次經驗不會被記錄

### 經驗分類
- **General Knowledge** (`knowledge-base/`)：通用流程、個人偏好、決策邏輯
- **Skill Experience** (`experience/`)：特定技能的參數、錯誤解法、成功案例

## 實戰範例

**用戶**：「幫我用 Python 寫一個爬蟲抓取股價。」

**Auto-Skill（內部運作）**：
1. 提取關鍵詞：Python、爬蟲、股價
2. 偵測技能調用：`skill-python-coding`
3. 讀取經驗庫：「上次抓股價被擋,改用 fake-useragent 解決」
4. AI 回覆：「沒問題,我會直接加上 fake-useragent 來避免被擋。」

**用戶**：「太棒了！這次一次就成功運行。」

**Auto-Skill（主動記錄）**：
「這次我們用 fake-useragent 成功解決了反爬問題。我想把這個『針對金融網站的 Header 設定技巧』記錄到你的 Python 經驗庫中,下次寫爬蟲時我可以自動套用。你覺得可以嗎？」
