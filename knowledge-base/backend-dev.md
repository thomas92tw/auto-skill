# 後端開發最佳實踐

此分類記錄後端開發相關的經驗和最佳實踐。

---

## 🔧 高效網頁抓取過濾器 (Path Depth & Content Blacklist)
**日期：** 2026-02-11
**情境：** 開發新聞監測或自動化抓取工具時，防止抓取到無效的首頁連結或非文章頁面（如搜尋、分類）。
**最佳實踐：**
1. **深度路徑校驗 (Path Depth)**：
   - 邏輯：檢查 URL 的 Path 段落數量。
   - 規則：排除根目錄 (HomePage)，要求 Path Segments 至少 >= 1。這能確保連結指向具體內容頁而非媒體門戶。
2. **內容屬性黑名單 (Content Blacklist)**：
   - 邏輯：掃描 URL 字串是否包含索引性質關鍵字。
   - 關鍵字清單：`search`, `category`, `tag`, `author`, `index`, `label`。
   - 效果：點擊報告連結時能 100% 直達全文文章，徹底杜絕無效導航連結。
3. **網域網格校驗**：
   - 與其使用黑名單排除垃圾網站，應優先使用 **權威白名單 (Allowlist)** 配合網域精準匹配，從源源頭確保資料品質。
